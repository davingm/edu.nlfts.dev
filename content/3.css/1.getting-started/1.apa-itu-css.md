## 1. Filosofi dan Definisi

Dalam arsitektur pengembangan web modern, terdapat tiga pilar utama yang menyusun sebuah aplikasi atau situs: **HTML** (Hypertext Markup Language) sebagai **lapisan struktur**, **CSS** (Cascading Style Sheets) sebagai **lapisan presentasi**, dan **JavaScript** sebagai **lapisan perilaku**.

CSS, secara formal, adalah *stylesheet language* yang digunakan untuk mengontrol tampilan dan tata letak (*layout*) dari dokumen yang ditulis dalam bahasa markup. Jika kita analogikan dengan pembangunan gedung, HTML adalah kerangka beton dan bata yang membentuk ruangan, sedangkan CSS adalah arsitektur interior, pemilihan warna cat, jenis lantai, pencahayaan, serta pengaturan posisi furnitur agar ruangan terasa nyaman dan fungsional.

Kekuatan utama CSS terletak pada kemampuannya untuk **memisahkan konten dari desain**. Pemisahan ini memberikan keuntungan fundamental dalam pemeliharaan kode (*maintainability*) dan efisiensi kerja, karena seorang *front-end engineer* dapat mengubah seluruh tampilan antarmuka hanya dengan memodifikasi satu atau beberapa file CSS, tanpa perlu menyentuh ribuan baris kode HTML.

## 2. Anatomi Aturan CSS: Sintaks Dasar

Sebelum kita terjun ke dalam implementasi, sangat penting untuk memahami unit terkecil dari CSS, yang disebut sebagai ***CSS Ruleset*** atau **Aturan CSS**.

Sebuah aturan CSS terdiri dari dua komponen utama: **Selektor** (memilih elemen mana yang akan dituju) dan **Blok Deklarasi** (berisi properti serta nilai yang akan diterapkan).

```css
/* 
   Ini adalah komentar dalam CSS.
   Komentar sangat penting untuk dokumentasi internal tim.
*/

selektor {
  properti-1: nilai-1;
  properti-2: nilai-2;
}
```

### 2.1. Selektor (Selector)
Selektor adalah pola (*pattern*) yang digunakan untuk mencocokkan elemen-elemen di pohon dokumen (*DOM Tree*). Pemahaman mendalam tentang selektor adalah kunci untuk menulis CSS yang efisien dan terhindar dari konflik.

### 2.2. Blok Deklarasi (Declaration Block)
Diawali dengan kurung kurawal buka (`{`) dan ditutup dengan kurung kurawal tutup (`}`). Di dalamnya terdapat satu atau lebih deklarasi yang dipisahkan oleh titik koma (`;`). Setiap deklarasi adalah pasangan `properti: nilai;`.

> **Peringatan Krusial**: Kelalaian dalam menambahkan titik koma (`;`) di akhir sebuah deklarasi, meskipun deklarasi tersebut adalah yang terakhir dalam blok, sering kali menjadi sumber *bug* halus yang menyebabkan deklarasi berikutnya tidak terbaca oleh *browser*.

## 3. Strategi Implementasi: 3 Metode Penerapan

Dalam praktik profesional, kita mengenal tiga pendekatan untuk menerapkan CSS ke dalam dokumen HTML. Pemilihan metode ini berdampak pada *performance*, *scalability*, dan *debugging*.

### 3.1. CSS Internal (Embedded Stylesheet)
Metode ini menempatkan kode CSS di dalam elemen `<style>` yang terletak di bagian `<head>` dokumen HTML.

```html
<!DOCTYPE html>
<html>
  <head>
    <style>
      body {
        background-color: #f8f9fa;
      }
    </style>
  </head>
</html>
```

| **Kelebihan** | **Kekurangan** |
| :--- | :--- |
| Cocok untuk halaman spesifik yang memiliki gaya unik | Tidak efisien untuk proyek berskala besar |
| Tidak memerlukan *request* HTTP tambahan | Menyulitkan *caching* di sisi klien |
| | Melanggar prinsip *Separation of Concerns* |

### 3.2. CSS Inline (Inline Styles)
Diterapkan langsung pada atribut `style` di dalam elemen HTML tertentu.

```html
<p style="color: #3498db; font-weight: bold;">
  Teks ini berwarna biru dan tebal.
</p>
```

| **Kelebihan** | **Kekurangan** |
| :--- | :--- |
| Prioritas tertinggi (mengalahkan semua aturan lain) | Sangat sulit dipelihara (*hard to maintain*) |
| Berguna untuk *dynamic styling* via JavaScript | Mencampur struktur dan presentasi (dianggap *anti-pattern*) |
| | Tidak dapat digunakan ulang (*reusable*) |

### 3.3. CSS External (Metode Standar Industri) — **Rekomendasi Utama**
Metode ini memisahkan semua kode CSS ke dalam file berekstensi `.css` terpisah, lalu dihubungkan (*linked*) ke dokumen HTML menggunakan elemen `<link>` di `<head>`.

```html
<!-- index.html -->
<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" href="./assets/styles/main.css">
  </head>
</html>
```

```css
/* assets/styles/main.css */
body {
  background-color: #f8f9fa;
}
```

| **Kelebihan** | **Kekurangan** |
| :--- | :--- |
| **Reusabilitas maksimal** (satu file CSS bisa dipakai oleh banyak halaman) | Memerlukan *request* HTTP tambahan (walaupun ini bisa diatasi dengan HTTP/2) |
| Memanfaatkan *browser caching* untuk meningkatkan kecepatan *load* halaman berikutnya | |
| Kode tetap bersih, terstruktur, dan mudah untuk *debugging* serta *versioning* | |
| Memungkinkan penggunaan *preprocessors* seperti SASS/SCSS di masa depan | |

> **Catatan Penting**: Dalam dokumentasi resmi framework besar seperti Next.js atau Nuxt.js, metode External selalu menjadi fondasi utama, yang kemudian dikembangkan lebih lanjut dengan sistem *CSS Modules* atau *Scoped CSS* untuk mengatasi masalah *global namespace collision*.

## 4. Mekanisme Cascading dan Spesifisitas

Nama "Cascading" bukanlah sembarang kata. Ia merujuk pada algoritma bawaan *browser* yang menentukan aturan mana yang berlaku ketika terdapat konflik (dua atau lebih aturan menunjuk ke elemen yang sama). Algoritma ini bekerja berdasarkan tiga pilar utama:

### 4.1. Asal-usul Stylesheet (Origin)
*Browser* mengenal tiga asal utama: **User-agent** (gaya bawaan browser), **User** (gaya yang ditentukan pengguna), dan **Author** (gaya yang ditulis oleh developer). Tentu saja, sebagai developer, gaya kita (Author) memiliki prioritas tertinggi kecuali pengguna menandai gaya mereka dengan `!important`.

### 4.2. Spesifisitas (Specificity)
Ini adalah inti dari logika CSS. Spesifisitas dihitung berdasarkan sebuah sistem nilai berbasis 4 kolom `(a, b, c, d)`:

- **Kolom A (Inline style)**: Apakah elemen memiliki atribut `style` langsung? Jika ya, nilainya `1, 0, 0, 0`.
- **Kolom B (ID)**: Jumlah selektor ID (misal `#header`).
- **Kolom C (Class, Pseudo-class, Attribute)**: Jumlah selektor *class* (`.btn`), *pseudo-class* (`:hover`), atau *attribute selector* (`[type="text"]`).
- **Kolom D (Elemen & Pseudo-elemen)**: Jumlah selektor elemen (`div`, `p`) atau *pseudo-elemen* (`::before`).

**Contoh Perhitungan:**

```css
/* Spesifisitas: 0, 1, 0, 1  (Berat: 100 + 1 = 101) */
#container p {
  color: black;
}

/* Spesifisitas: 0, 0, 2, 0  (Berat: 10 + 10 = 20) */
.card .title {
  color: red;
}
```
Dalam kasus di atas, meskipun aturan `#container p` berada di atas, ia tetap menang karena spesifisitasnya (101) jauh lebih tinggi daripada `.card .title` (20).

### 4.3. Urutan Kemunculan (Order of Appearance)
Jika dua aturan memiliki spesifisitas yang sama persis, maka aturan yang dideklarasikan **paling akhir** (paling bawah) dalam file CSS akan diterapkan.

> **Larangan Umum**: Penggunaan `!important` dianggap sebagai *code smell* karena merusak alur kaskade alami. Dalam tim besar, penggunaannya sering disepakati hanya untuk *utility classes* atau *overrides* sistemik, bukan untuk logika komponen harian.

## 5. Eksplorasi Selektor Dasar dan Turunannya

Meskipun ini adalah halaman fundamental, penting untuk melihat lebih jauh daripada sekadar `div` atau `.class`.

| Jenis Selektor | Contoh | Penjelasan Formal |
| :--- | :--- | :--- |
| **Universal** | `* { box-sizing: border-box; }` | Mencocokkan **semua** elemen. Biasanya digunakan untuk *reset CSS*. |
| **Tipe (Type)** | `article { ... }` | Mencocokkan elemen berdasarkan nama simpul (*node name*). |
| **Kelas (Class)** | `.navbar { ... }` | Mencocokkan elemen berdasarkan atribut `class`. Nilai spasial yang sangat fleksibel. |
| **ID** | `#submit-btn { ... }` | Mencocokkan elemen berdasarkan atribut `id`. Harus **unik** dalam satu halaman. |
| **Grup (Grouping)** | `h1, h2, h3 { ... }` | Menggabungkan beberapa selektor untuk menerapkan deklarasi yang sama. |
| **Turunan (Descendant)** | `article p { ... }` | Mencocokkan elemen `<p>` yang berada **di mana saja** di dalam `<article>`. |
| **Anak Langsung (Child)** | `article > p { ... }` | Hanya mencocokkan `<p>` yang merupakan anak **langsung** (level 1) dari `<article>`. |

## 6. Eksplorasi Properti Inti (Core Properties)

Pada tahap awal, Anda perlu menguasai kelompok properti visual dasar yang paling sering muncul di 90% antarmuka web.

### 6.1. Tipografi dan Teks
Mengatur tampilan huruf sangat krusial untuk *readability*.
- `color`: Menerima nilai *keyword* (`red`), *hexadecimal* (`#ff0000`), *RGB* (`rgb(255,0,0)`), atau *HSL*.
- `font-size`: Gunakan `rem` (relatif terhadap *root font size*) untuk aksesibilitas. Hindari `px` secara berlebihan.
- `font-family`: Selalu akhiri dengan *fallback generic family*, misal: `font-family: 'Inter', 'Segoe UI', sans-serif;`.
- `text-align`: Mengontrol perataan horizontal (kiri, kanan, tengah, atau *justify*).

### 6.2. Dimensi dan Spasial
Mengatur ukuran dan jarak.
- `width` & `height`: Bisa berupa nilai pasti (`100px`), relatif (`50%`), atau otomatis (`auto`).
- `margin`: Ruang **di luar** batas elemen (mendorong elemen lain menjauh).
- `padding`: Ruang **di dalam** batas elemen (antara *border* dan konten).
- `border`: Menggabungkan `border-width`, `border-style`, dan `border-color`.

## 7. Studi Kasus Praktis: Membangun Kartu Profil

Mari kita terapkan semua konsep fundamental di atas ke dalam sebuah komponen nyata.

**Struktur HTML:**
```html
<!DOCTYPE html>
<html lang="id">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <link rel="stylesheet" href="style.css">
  <title>Kartu Profil</title>
</head>
<body>

  <div class="profile-card">
    <img src="avatar.jpg" alt="Foto Profil" class="profile-card__image">
    <h2 class="profile-card__name">Dr. Andi Wijaya</h2>
    <p class="profile-card__title">Front-end Architect</p>
    <p class="profile-card__description">
      Spesialis dalam sistem desain dan performa web.
    </p>
  </div>

</body>
</html>
```

**Implementasi CSS (style.css) Eksternal:**
```css
/* 1. Reset universal untuk menghilangkan margin bawaan browser */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box; /* Mengubah cara perhitungan lebar elemen */
}

/* 2. Styling badan dokumen */
body {
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
  background-color: #f1f5f9;
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh; /* Memenuhi tinggi layar */
}

/* 3. Desain komponen utama menggunakan metodologi BEM (Block Element Modifier) */
.profile-card {
  background-color: #ffffff;
  width: 320px;
  padding: 24px;
  border-radius: 16px;
  box-shadow: 0 10px 25px rgba(0, 0, 0, 0.05);
  text-align: center;
  transition: transform 0.2s ease; /* Animasi halus saat hover */
}

.profile-card:hover {
  transform: translateY(-4px); /* Efek mengangkat saat disorot */
}

/* 4. Selektor kelas untuk elemen di dalam blok */
.profile-card__image {
  width: 96px;
  height: 96px;
  border-radius: 50%; /* Membuat gambar menjadi lingkaran sempurna */
  border: 4px solid #e2e8f0;
  margin-bottom: 16px;
}

.profile-card__name {
  color: #0f172a;
  font-size: 1.25rem; /* 20px jika root font-size 16px */
  font-weight: 600;
  margin-bottom: 4px;
}

.profile-card__title {
  color: #3b82f6;
  font-size: 0.875rem;
  font-weight: 500;
  margin-bottom: 12px;
}

.profile-card__description {
  color: #475569;
  font-size: 0.875rem;
  line-height: 1.5; /* Memberi jarak antar baris teks */
}
```
